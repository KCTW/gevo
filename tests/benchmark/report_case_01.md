# 測試報告 (TestCase: CASE-01 理財 App)

## 領域實體與狀態
- **實體**: 帳目 (Entry) [草稿 -> 確認 -> 已歸檔]、帳戶 (Account)、類別 (Category)
- **狀態**: 同步中、本地緩存、同步衝突

## 技術需求 (Zero-Shot)
- **US-01**: 實作本地優先 (Local-first) 儲存架構，確保離線可用。
- **US-02**: 介接銀行 API (Open Banking) 或簡訊擷取自動記錄支出。
- **US-03**: 實作基於 OAuth2 的安全驗證。

## 邊界條件
- 斷網期間的數據併發修改。
- 銀行 API 認證過期。
